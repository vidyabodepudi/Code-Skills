# /spec — Define Phase

Activate the Define phase of the CodeHands lifecycle.

## Skills Activated
1. `codehands:brainstorming` — Explore the idea with Socratic dialogue, present 2-3 approaches, get design approval
2. `codehands:spec-driven-development` — Formalize requirements with RFC 2119 keywords and testable acceptance criteria

## Process
1. Start with `codehands:brainstorming` — explore context, ask questions, propose approaches
2. After design approval, transition to `codehands:spec-driven-development`
3. Write the spec to `docs/specs/YYYY-MM-DD-<topic>-spec.md`
4. Get user sign-off on the spec

## Hard Gate
Do NOT write any code until the spec is approved.
